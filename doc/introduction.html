Using facet for Generalizability Theory

Introduction to facet

The facet package provides comprehensive tools for conducting Generalizability Theory (G-theory) analyses using variance component models. It supports three estimation backends: frequentist (lme4), Bayesian (brms), and method of moments (mom/ANOVA-based). The package provides a unified interface for univariate and multivariate designs, with full support for G-studies, D-studies, and coefficient calculations with uncertainty quantification.

Note: The tidy() and glance() methods require the broom package to be loaded. The examples below use library(broom) to ensure these functions are available.

Overview

Generalizability Theory provides a framework for quantifying the reliability of measurements across multiple sources of error. The key concepts are:

  • G-studies: Estimate variance components associated with different facets of measurement (e.g., items, raters, occasions)
  • D-studies: Compute generalizability (G) and dependability (D/Phi) coefficients for different measurement designs

The facet package supports multiple backends:

  • lme4: For frequentist estimation using linear mixed effects models (default for univariate models)
  • brms: For Bayesian estimation using Stan (required for multivariate models)
  • mom: For method of moments (ANOVA-based) estimation. Works best with balanced designs and provides confidence intervals using asymptotic approximations.

Basic G-Study

A G-study estimates the variance components for each facet in your measurement design. The formula syntax follows lme4 conventions, specifying random effects with (1 | facet).

In this example, we use the brennan dataset, which contains scores for 10 persons across 3 tasks and 12 raters in a fully crossed design (\(p \times t \times r\)).

G-Study with Confidence Intervals

For the lme4 backend, you can request confidence intervals using the ci_method parameter:

  • "none" (default): No confidence intervals
  • "profile": Profile likelihood confidence intervals (more accurate, slower)
  • "boot": Parametric bootstrap confidence intervals (most accurate, slowest)
# G-study with profile likelihood confidence intervals
g_profile <- gstudy(
  Score ~ (1 | Person) + (1 | Task),
  data = brennan,
  ci_method = "profile"
)

Bayesian Estimation with brms

For Bayesian estimation, use the brms backend. This provides:

  • Posterior distributions for variance components
  • Credible intervals (highest posterior density intervals)
  • Full Bayesian inference including convergence diagnostics
# Bayesian G-study with brms
# Note: Requires brms and Stan to be installed
g_bayes <- gstudy(
  Score ~ (1 | Person) + (1 | Task),
  data = brennan,
  backend = "brms"
)

print(g_bayes)
summary(g_bayes)

The brms backend provides additional diagnostic information like Rhat (Gelman-Rubin diagnostic) and Effective Sample Size (ESS) values.

D-Study

A D-study uses the G-study results to compute reliability coefficients for different measurement designs. The key coefficients are:

  • G coefficient: Generalizability coefficient for relative decisions (comparing individuals’ rankings)
  • D coefficient: Dependability (\(\Phi\)) coefficient for absolute decisions (comparing to a standard)

Specifying Object of Measurement

By default, the first facet in the formula is considered the object of measurement (typically “Person”). You can specify this explicitly or modify the universe/error structure:

# D-study with custom universe specification
# The object of measurement is Person
d_custom <- dstudy(g, n = list(Task = 3, Rater = 4), 
                   universe = c("Person", "Person:Task"))

The universe parameter accepts multiple formats:

  • Character string: "Person"
  • Character vector: c("Person", "Person:Task")
  • Formula: ~ Person + Person:Task

Note: The object of measurement (first facet from the G-study) is always included in the universe. If you specify a universe without including the object, a warning is issued and the object is added automatically.

The universe score variance is calculated as follows: - The object component is not scaled. - Non-object universe components are scaled by their corresponding sample sizes. - For example, if the object is “Person” and universe includes “Person:Task”, then: - \(\sigma^2(\text{Universe}) = \sigma^2(P) + \sigma^2(PT) / n_t\)

Advanced D-Study: Custom Error Specification

You can customize which variance components contribute to the error term using the error parameter:

# D-study with custom error specification
# Using only interaction terms as error sources
d_error <- dstudy(
  g,
  n = list(Task = 3, Rater = 4),
  error = c("Person:Task", "Person:Rater")
)

Posterior-Based D-Study Coefficients

For Bayesian G-studies, you can compute D-study coefficients using the full posterior distributions:

# D-study with posterior estimation (requires brms backend)
# Note: This uses full posterior to compute coefficients with uncertainty
d_post <- dstudy(
  g_bayes,
  n = list(rater = 3),
  estimation = "posterior"
)

print(d_post)

# Access posterior distributions
names(d_post$posterior)

If you request posterior estimation but the G-study was not fit with brms, the function will automatically refit:

# D-study with posterior from lme4 gstudy
d_post2 <- dstudy(
  g,
  n = list(rater = 3),
  estimation = "posterior"
)

Multivariate Analysis

For multivariate designs (multiple outcome variables), use the brms backend syntax with mvbind().

# Create multivariate data by spreading the tasks
brennan_wide <- brennan %>%
  pivot_wider(names_from = Task, values_from = Score, names_prefix = "Task")

# Multivariate G-study (automatically uses brms)
g_mv <- gstudy(
  mvbind(Task1, Task2, Task3) ~ (1 | Person) + (1 | Rater),
  data = brennan_wide
)

summary(g_mv)

# D-study for multivariate models
d_mv <- dstudy(g_mv, n = list(Rater = 4))
print(d_mv)
